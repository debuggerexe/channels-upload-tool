# 微信视频号自动上传工具

自动化发布视频到微信视频号平台，支持多种数据源（Notion、飞书、本地文件）。

## 功能特性

- **混合模式（Notion/飞书）**
  - 从 Notion/飞书 数据库获取视频元数据（标题、描述、标签等）
  - 优先匹配本地 `videos/` 文件夹中的视频，无本地则从云端下载
  - 智能本地缓存，避免重复下载

- **智能处理**
  - 智能文件名匹配（自动去除日期前缀）
  - 短标题自动清理（只保留中文字符，**超过16字自动截断**）
  - 封面智能裁剪（高图自动裁剪为3:4比例）
  - 自动声明原创、自动添加合集
  - **并行上传优化**（上传期间填写表单，节省时间）

- **批量上传**
  - 按发布日期自动排序
  - 失败自动重试（网络波动时自动重试 API 请求）
  - 上传成功后自动清理临时文件，本地缓存保留

## 项目架构

```
📁 channels-upload-tool/
├── 📄 upload_wechat_videos.py    # 主入口脚本
├── 📄 conf.py                     # 全局配置
├── 📄 config.example.json         # 配置模板（复制为 config.json 后填写）
│
├── 📁 data_sources/               # 数据源层（抽象基类 + 多实现）
│   ├── 📄 data_source.py          # VideoDataSource 抽象基类
│   ├── 📄 VideoInfo (dataclass)   # 统一视频信息数据结构
│   ├── 📄 notion_data_source.py   # Notion 数据源（混合模式）
│   ├── 📄 feishu_data_source.py   # 飞书数据源（混合模式）
│   └── 📄 local_data_source.py    # 本地数据源（txt配置）
│
├── 📁 uploader/                   # 上传器层
│   └── 📁 tencent_uploader/       # 微信视频号上传器
│       └── 📄 main.py             # Playwright 自动化上传核心
│
├── 📁 utils/                      # 工具层
│   ├── 📄 output_utils.py         # 终端输出模板（5阶段统一输出）
│   ├── 📄 text_utils.py           # 文本处理（短标题清理、描述组装等）
│   ├── 📄 match_utils.py          # 文件匹配（本地视频查找、封面匹配）
│   ├── 📄 download_utils.py       # 云端文件下载
│   ├── 📄 cover_cropper.py        # 封面裁剪工具
│   └── 📄 files_times.py          # 文件时间处理
│
├── 📁 videos/                     # 本地视频文件夹（gitignored）
├── 📁 temp/                       # 临时下载文件夹（自动清理）
├── 📁 cookies/                    # Cookie 存储（gitignored）
└── 📁 logs/                       # 日志文件（gitignored）
```

**架构设计原则：**
- **数据源抽象**：`VideoDataSource` 基类统一接口，支持多数据源无缝切换
- **混合模式**：优先本地文件，无本地则从云端下载，兼顾速度可靠性
- **终端输出统一**：5阶段标准输出（数据获取→状态筛选→本地扫描→匹配结果→上传汇总）
- **配置分离**：敏感信息放 `config.json`（gitignored），模板放 `config.example.json`

## 快速开始

```bash
source .venv/bin/activate && python upload_wechat_videos.py --mode cloud --no-interactive
```

### 上传流程

1. 从 Notion 获取待发布视频列表（按发布时间排序）
2. 优先检查本地 `videos/` 文件夹，如有匹配则直接使用
3. 无本地文件则从 Notion 云端下载到 `temp/`
4. 上传视频到微信视频号，期间并行填写表单（标题、合集、原创声明等）
5. 上传完成后清理 `temp/` 临时文件，本地 `videos/` 文件保留

## 混合模式（Notion）

### Notion 数据库字段

| 字段 | 类型 | 说明 | 必填 |
|------|------|------|------|
| `Name` | `Title` | 视频名称（用于标识），同时用于匹配本地视频文件名 | ✅ |
| `短标题` | `Rich Text` | 视频号短标题（16字以内），只保留中文字符 | ✅ |
| `标题` | `Rich Text` | 视频号主标题（显示在视频下方） | 可选 |
| `描述` | `Rich Text` | 视频描述区内容 | 可选 |
| `标签` | `Rich Text` | 话题标签（如 `#话题1 #话题2`） | 可选 |
| `合集` | `Multi-select` | 视频合集名称列表（可多选） | 可选 |
| `发布日期` | `Date` | 发布日期时间，可只填日期（默认 09:00）或完整时间 | ✅ |
| **视频** | **Files & media** | **视频文件附件** | ✅ |
| **封面** | **Files & media** | **封面图附件** | 可选 |
| **声明原创** | **Checkbox** | **是否声明原创** | 可选（默认true） |
| **封面裁剪** | **Select** | **top/middle/bottom** | 可选（默认middle） |
| **位置** | **Select** | **不显示位置/平台默认** | 可选（默认平台默认） |
| **发布状态** | **Select** | **待发布/已发布/发布失败** | 混合模式必填 |
| **发布方式** | **Select** | **定时发布/保存草稿** | 可选 |

**字段说明：**
- **发布状态**（混合模式）：控制视频上传流程
  - `待发布`：脚本会筛选并上传这些视频
  - `已发布`：上传完成后脚本自动更新为此状态
  - `发布失败`：上传或下载失败时脚本自动更新为此状态
  - 只有 `待发布` 和 `发布失败` 的视频会被处理（发布失败会自动重试）
  - 上传成功后，无论之前是待发布还是发布失败，都会更新为 `已发布`
- **发布方式**（混合模式）：控制视频的发布方式
  - `定时发布`：按 `发布日期` 字段的时间定时发布
  - `保存草稿`：保存到草稿箱，不立即发布
  - 优先级逻辑：
    - 发布方式 = 保存草稿 → 保存草稿
    - 发布方式 = 定时发布 + 有发布日期 → 定时发布
    - 发布方式 = 定时发布 + 无发布日期 → 降级为保存草稿
    - 发布方式 = 空 + 有发布日期 → 默认定时发布
    - 发布方式 = 空 + 无发布日期 → 默认保存草稿
- **Name**：视频名称（用于标识），同时用于匹配本地视频文件
- **短标题**：视频号顶部显示的短标题（16字以内）
  - 只保留中文字符（自动去掉英文、数字、符号）
  - 不足6字自动补空格
  - 超过16字**强行截断**（直接取前16字，不保留完整词义）
- **标题**：视频下方显示的主标题
- **描述**：视频描述区内容
- **标签**：话题标签，会自动追加到描述区后面
- **合集**：视频合集名称列表（支持多选）
  - Notion 中使用 `Multi-select` 类型，可添加多个合集
  - 上传时自动点击"选择合集"按钮
  - **匹配规则**：Notion 中的任意合集名称与视频号平台的任意合集**完全匹配**即可自动选择
  - 全部未匹配到会提示在视频号后台手动创建
- **发布日期**：发布日期时间
  - 可填写完整时间（如 `2024-01-15 14:30`）
  - 也可只填日期（如 `2024-01-15`），默认使用 **09:00** 作为发布时间
  - 时间从 Notion 的 `发布日期` 字段读取，不再使用 `config.json` 中的配置
- **位置**：控制视频的位置显示
  - `平台默认`（默认）：跳过位置设置，保持视频号平台默认加载的内容
  - `不显示位置`：点击位置区域，选择"不显示位置"

**视频号描述区格式：**

格式：`标题 + 换行 + 描述 + 换行 + 标签`

| 部分 | 来源字段 | 说明 |
|------|----------|------|
| 标题 | `标题` 字段 | 视频号主标题，显示在描述区第一行 |
| 描述 | `描述` 字段 | 视频详细描述内容 |
| 标签 | `标签` 字段 | 以 `#` 开头的话题标签 |

**标签去重逻辑：**
- 如果**描述字段的最后一行以 # 开头**（表示已包含话题标签），则不再追加标签字段
- 如果描述字段中没有标签行，且有标签字段，则将标签追加到描述后面

**示例1（描述已含标签）：**
```
标题: "啊啊啊"
描述: "啊啊啊啊啊...\n#标签1 #标签2"
标签: "#标签1 #标签2"
```
→ 最终描述区内容：
```
啊啊啊
啊啊啊啊啊...
#标签1 #标签2
```
（标签字段被忽略，因为描述中已有标签）

**示例2（描述不含标签）：**
```
标题: "啊啊啊"
描述: "啊啊啊啊啊..."
标签: "#标签1 #标签2"
```
→ 最终描述区内容：
```
啊啊啊
啊啊啊啊啊...
#标签1 #标签2
```

### 网络优化建议（国内访问）

Notion 在国内访问可能较慢或不稳定，项目已做以下优化：

1. **下载重试机制** - 视频文件自动重试 3 次，封面文件重试 2 次
2. **指数退避** - 失败后等待时间递增（2秒、4秒、8秒）
3. **封面容错** - 封面下载失败不影响视频上传（使用视频默认封面）
4. **批量并发** - 多个视频同时下载提高效率

**建议：**
- 上传前确保网络稳定
- 视频文件较大时可分批上传（修改 Notion 状态筛选）
- 如频繁失败，建议在非高峰期使用或配合代理

或者设置环境变量：

```bash
export NOTION_API_TOKEN='your_token'
```

## 混合模式（飞书）

项目同时支持 **飞书多维表格** 作为数据源，与 Notion 混合模式完全兼容。

```bash
source .venv/bin/activate && python upload_wechat_videos.py --mode feishu --no-interactive
```

### 飞书配置获取步骤

1. **创建飞书应用**
   - 访问 [飞书开放平台](https://open.feishu.cn)
   - 创建企业自建应用
   - 获取 **App ID** 和 **App Secret**

2. **添加应用权限（批量导入）**
   - 进入应用详情 → 权限管理
   - 点击「批量导入/导出权限」按钮
   - 粘贴以下 JSON 配置：
   ```json
   {
     "scopes": {
       "tenant": [
         "base:record:read",
         "base:record:write",
         "base:file:read",
         "bitable:app",
         "bitable:app:readonly",
         "drive:file:download",
         "drive:file:readonly"
       ],
       "user": []
     }
   }
   ```
   - 点击「下一步」→「确认新增权限」→「申请开通」
   - **发布应用版本**使权限生效（版本管理与发布 → 创建版本 → 保存并发布）

3. **添加应用为表格协作者（关键步骤）**
   - 打开目标多维表格
   - 点击右上角「分享」按钮（或「...」→「分享」）
   - 在协作者搜索框中输入你的应用名称（如「uploadbot」）
   - 选择应用并赋予**「可编辑」或「可查看」权限**
   - 点击「发送」或「确认」
   - ⚠️ **如果不执行此步骤，应用无法访问你的个人表格，会报 `RolePermNotAllow` 错误**

4. **获取多维表格Token**
   - 打开目标多维表格
   - 从URL中复制 `app_token`（如 `CqZ5b6kQia...`）
   - 记录 `table_id`（如 `tblm3I9JaZ9JeIs7`）

### 飞书混合模式

由于飞书 API 权限限制，个人用户可能无法直接通过 API 读取多维表格记录。混合模式提供优雅的降级方案：

**工作原理：**
1. 尝试从飞书读取视频配置（标题、描述、发布日期等）
2. 从本地 `videos/` 文件夹匹配视频文件
3. 如果飞书 API 权限不足，自动回退到本地模式

**优势：**
- 无需复杂的权限申请和审核
- 配置管理在飞书，视频文件在本地
- 自动回退机制，确保上传不中断

## 代码架构优化

### 统一上传流程

**重构前的问题：**
- 本地模式和云端模式有 100+ 行重复代码
- 上传逻辑分散在两个方法中
- 维护困难，容易遗漏功能同步

**重构后的结构：**

```python
# 公共方法（统一处理）
_upload_single_video()     # 单视频上传（含重试、封面处理）
_move_to_published()       # 文件归档到 published 目录

# 模式入口（仅数据组装和流程控制）
upload_all_videos()        # 本地模式 → 调用公共方法
upload_videos_from_source() # 云端模式 → 调用公共方法
```

**优化收益：**
- 代码量减少约 60%
- 两种模式功能完全一致（发布方式、短标题截断、封面处理）
- 维护简单，只需修改一处即可同步到两种模式

### 封面处理优化

**问题：** 视频上传完成后，封面预览元素可能还未就绪，导致封面处理失败。

**解决：** 添加封面预览等待逻辑，最多等待 10 秒确保封面元素加载完成后再处理。
